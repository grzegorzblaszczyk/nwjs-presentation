Problem:

Jak zbudować aplikację typu standalone ?
  • GTK+ - Gimp Toolkit
  • QT - dawniej Quasar Technologies, potem Trolltech i Qt Software
  • WxWidgets
  • Java Swing / AWT
  • ttk - Tk themed widgets - Python
  • MFC (Microsoft Foundation Classes) - Windows API

Marzenie:

aplikacja standalone == webaplikacja
  • HTML 5
  • CSS3
  • JavaScript
  • node.js
  • OpenGL => WebGL
  • ma działać offline

Rozwiązanie:

NW.js

0.12.0
(dawniej node-webkit)

node.js + Chromium

Chromium

  • przeglądarka od Google - open-source
  • web-app runtime
    • 									
<html>
  <head>
    <link rel="stylesheet" href="css/style.css">
    <script src="js/main.js"></script>
  </head>
  <body>
    <div id="foobar"></div>
  </body>
</html>
  • działa na Windows / Linux / Mac OS X / BSD*
  • wielkość aplikacji = 35 MB + wielkość aplikacji webowej (images/CSS/JS)

node.js

  • wszystkie biblioteki native działają (net, fs, Jade, underscore, ...)
  • wszystkie biblioteki JS działają (jQuery, bootstrap, Angular, ...)
  • DOM, window, document, ...
  • jeden zespół programistów

NW.js

  • wspierany i rozwijany przez firmę Intel
  • oprogramowanie open-source od 7.12.2011
  • github.com/nwjs/nw.js
  • wsparcie dla cocos2d-html
  • audio / video
  • akceleracja GPU

HOW TO

  1. HTML / JS / CSS / images / ... =(compress)=> App.nw
  2. cat nw.js App.nw > App
  3. App => App.exe / App.app
